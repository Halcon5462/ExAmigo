CONTRIBUTING

1. ОБЩИЕ ПРАВИЛА

Спасибо за вклад в проект.

Перед началом работы:

* Убедитесь, что локальная ветка develop актуальна
* Вся работа ведётся от ветки develop
* Строго соблюдайте правила именования веток и коммитов

---

2. СТРАТЕГИЯ ВЕТОК

Основные ветки:

* main — стабильная версия (production)
* develop — основная ветка разработки

Все рабочие ветки создаются от develop.

Типы веток:

* features/<название_механики>
  Для разработки нового функционала

* data/<название_области>
  Для изменений, связанных с базой данных

* style/<блок_или_страница>
  Для правок интерфейса и стилей

Примеры:

* features/variant-generation
* data/user-progress
* style/login-page

---

3. ФОРМАТ КОММИТОВ

Каждый коммит ДОЛЖЕН соответствовать формату:

type(scope) #task_number описание

Где:

* type — тип изменения
* scope — часть проекта (например: backend, taskBank)
* task_number — номер задачи (если есть)
* описание — краткое описание в инфинитиве

---

4. ТИПЫ КОММИТОВ

Допустимые префиксы:

* feat      — новая функциональность или изменение механики
* fix       — исправление ошибок
* data      — изменения базы данных
* refactor  — рефакторинг без изменения логики
* doc       — изменения документации

---

5. ПРИМЕРЫ КОММИТОВ

feat(taskBank) #11 Добавить генерацию варианта
fix(backend) #23 Исправить ошибку авторизации
data(users) #45 Добавить поле прогресса
refactor(api) Упростить обработку запросов
doc(readme) Обновить документацию

---

6. ТРЕБОВАНИЯ К КОММИТАМ

* Формулировки должны быть краткими и понятными
* Один коммит = одно логическое изменение
* Нельзя смешивать несвязанные изменения в одном коммите

---

7. ПРОЦЕСС РАЗРАБОТКИ

* Обновить ветку develop
* Создать новую ветку от develop
* Внести изменения
* Сделать коммит по правилам
* Запушить ветку
* Создать Merge Request в develop

---

8. ДОПОЛНИТЕЛЬНО

* Перед пушем убедитесь, что проект собирается и работает
* Соблюдайте существующий стиль кода
* Не вносите лишние изменения вне своей задачи

---
