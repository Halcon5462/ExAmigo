## Описание проекта

Платформа для формирования и прохождения комплектов заданий для подготовки к экзаменам.
Система позволяет решать задания, отслеживать прогресс пользователя и формировать новые наборы заданий на основе результатов.

## Техническое задание

### Проблема

Тренажёры для подготовки к экзаменам часто слабо подстраиваются под конкретную программу и уровень ученика, а расширенные возможности и аналитика нередко доступны только по подписке. По отдельным предметам и форматам заданий часто отсутствуют качественные тренажёры, из-за чего подготовка осуществляется по разрозненным источникам.

### Цель проекта

Создать платформу, которая:

* формирует комплекты заданий для подготовки к экзаменам по выбранному предмету;
* проверяет и оценивает выполнение заданий;
* фиксирует прогресс пользователя;
* генерирует новые комплекты заданий с учётом уровня подготовки пользователя.

### Тематика

Подготовка к экзаменам по школьным и университетским предметам.

## Запуск проекта

### Быстрый запуск (рекомендуется)

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File start.ps1
```

Linux / MacOS / WSL:

```bash
chmod +x start.sh
./start.sh
```

## Ручной запуск

### Backend

```bash
cd backend
python3 -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / MacOS
source .venv/bin/activate

pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

## Первый запуск

Установить зависимости backend:

```bash
pip install -r requirements.txt
```

Установить зависимости frontend:

```bash
npm install
```

Настроить переменные окружения frontend/src/utils/api.js:

```
const API_URL = 'http://127.0.0.1:8000/api';
```

## Развертывание проекта

### Быстрый запуск (development)

```bash

```

---

### Что делает start.sh

Скрипт автоматически:

**Backend:**
- создаёт виртуальное окружение
- устанавливает зависимости
- применяет миграции
- запускает Django server

**Frontend:**
- устанавливает зависимости npm
- запускает dev-сервер Vite

## Структура backend

При добавлении новой модели:

1. models.py — описание модели
2. serializers.py — сериализация данных
3. views.py — бизнес-логика
4. urls.py — маршруты API
5. подключение в main urls.py

Рекомендуется использовать докстринги.
Пример реализации: модель Task.

## Структура frontend

* src/pages — страницы приложения
* src/components — переиспользуемые компоненты
* src/App.jsx — маршрутизация
* public — статические файлы

## Документация

Документация по backend-проекту находится в папке `docs`.

Сборка HTML:

- `cd docs`
- `make html` (или `sphinx-build -b html . _build/html`)

Готовый сайт будет доступен в `docs/_build/html/index.html`.

Документация автоматически собирается и деплоится на GitHub Pages при пуше в основную ветку через GitHub Actions.

В `backend/requirements.txt` уже указаны зависимости Sphinx.


## Философия и ключевые фишки проекта

### Идея проекта

Мы решили объединить игровую концепцию, регулярное решение заданий и современные технологии, чтобы сделать процесс подготовки более вовлекающим и эффективным.

### Соревновательный режим

Реализован режим соревнований, реализованный через WebSocket.

Он решает сразу несколько задач:

* создаёт соревновательный дух;
* повышает мотивацию к обучению;
* увеличивает вовлечённость пользователей;
* способствует органическому росту аудитории через социальное взаимодействие.


### Автоматическая генерация заданий

Многим пользователям не хватает режима, в котором можно просто решать задания без дополнительных действий.

Мы реализовали автоматическую генерацию вариантов на основе статистики пользователя, что позволяет:

* непрерывно тренироваться;
* не отвлекаться на выбор заданий;
* адаптировать сложность под уровень пользователя.


### Система статистики

Во время подготовки часто сложно понять, какие темы уже освоены, а какие требуют повторения.

Для решения этой проблемы реализована система статистики:

* отслеживание успешности по каждому типу заданий;
* визуализация прогресса;
* подробная аналитика по каждому номеру заданий.

Это позволяет пользователю осознанно подходить к подготовке и корректировать слабые места.


### Обучение через нейросети

Дополнительно в проекте используется интеграция с DeepSeek API.

Реализованы следующие механики:

* подсказка (намёк на решение);
* идея решения;
* полное решение;
* режим “задать вопрос ИИ”.

Каждая функция направлена на то, чтобы превратить процесс решения задач в обучающий процесс.

Также используется внутренняя валюта, которую пользователь получает за выполнение заданий и достижения.


### Итог

Представленные механики являются ключевыми для проекта и формируют его основную ценность —
сочетание обучения, аналитики и игровой мотивации в одном сервисе для подготовки к экзаменам.
