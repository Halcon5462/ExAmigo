# Проект "Сервис для подготовки к экзаменам"

## Запуск проекта

### Регулярный запуск
- cd backend
- - . ./.venv/bin/activate - если Linux/MacOs
- - . ./.venv/Scripts/activate - если Windows
- python3 manage.py runserver
- Открыть второй терминал
- cd ../
- cd frontend
- npm run dev
- перейти по ссылке из терминала frontend

### Первый запуск
- cd backend
- python3 -m venv .venv - создание виртуального окружения
- - . ./.venv/bin/activate - активация виртуального акружения если Linux/MacOs
- - . ./.venv/Scripts/activate - активация виртуального акружения если Windows
- pip install -r requirements.txt - скачивание нужных библиотек для бэкенда
- python manage.py runserver - запуск бэкенда
- Открыть второй терминал
- cd ../
- cd frontend
- npm i - скачивание всех нужных библиотек для фронтенда
- npm run dev - запуск фронтенда
- перейти по ссылке из терминала frontend
- в файле frontend/.env прописан адрес бэкенда, убедитесь совпадает ли он с вашим из терминала backend
- в случае различия перезапишите его на версию из терминала

## Работа с проектом

### "Админка"
- Ссылка для админки находится в терминале, где запущен backend. 
- Надо перейти по ней и добавить /admin
- Пароль и почту можно узнать у сеньора

### Структура backend

- При создании новой моделе необходимо написать ее код в models.py ее приложения 
- Позже создать сериалайзер в serializer.py
- Позже добавить функции для объектов этой модели в views.py
- Добавить необходимые ссылки в urls.py 
- Занести их в главный urls.py в папке backend
- Есть уже готовый пример - модель Task
- Необходимо писать докстринги 
- Вся работа идет в соответсвующем приложении

### Структура frontend

- Все медиа хранятся в public/
- Все остальные программы находятся в src
- Страницы в src/pages
- Компоненты для страниц в src/components
- Маршрутизация происходит в src/App.jsx
